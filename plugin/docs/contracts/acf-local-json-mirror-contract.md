# ACF Local JSON Mirror and Debug Export Contract

**Document type:** Contract for the plugin’s ACF field-group mirror and debug export layer.  
**Governs:** Mirror generation, debug export records, diff summary, and export-bundle inclusion (spec §20.3, §20.4, §20.15, §52, §59.13; Prompt 224).  
**Reference:** Master Specification §20, §52, §58.4, §58.5, §59.13.

---

## 1. Purpose and scope

- **Not the normal user workflow.** Manual JSON import per page or per template is not required. The plugin registry and code-defined programmatic registration remain the **source of truth**.
- **Mirror and debug export** support:
  - Source control visibility of programmatically defined field groups
  - Support diagnostics and environment comparison
  - Recovery tooling and regeneration/repair integration
- **Bounded and internal/debug:** Artifacts are clearly labeled; no shift to ACF local JSON as primary authority. No general-purpose field import wizard for ordinary users; no uncontrolled admin sync-from-file behavior.

---

## 2. Authorities and data shapes

### 2.1 Authorities

- Registry-defined field blueprints (section_key, section_version, fields)
- Deterministic key conventions (Field_Key_Generator: `group_aio_{section_key}`, field keys)
- Version-aware field-group definitions (`_aio_section_key`, `_aio_section_version`)
- Export/restore contracts (Export_Bundle_Schema, optional category `acf_field_groups_mirror`)

### 2.2 acf_local_json_manifest

Generated by `ACF_Local_JSON_Mirror_Service::generate_mirror_to_directory()` or `get_manifest_without_writing()`.

| Key             | Type     | Description |
|-----------------|----------|-------------|
| schema_version  | string   | Manifest schema version (e.g. `"1"`) |
| mirror_version  | string   | Mirror format version |
| generated_at    | string   | ISO 8601 UTC timestamp |
| plugin_version  | string   | Plugin version at generation time |
| source          | string   | Always `"registry"` |
| group_keys      | list     | List of group keys (e.g. `group_aio_st01_hero`) |
| files           | list     | Entries: `group_key`, `section_key`, `section_version`, `path_relative` |
| label           | string   | Human-readable label (internal/debug) |
| error           | bool     | Optional; true when generation failed |
| message         | string   | Optional; error message |

### 2.3 field_group_debug_export_record

Generated by `ACF_Field_Group_Debug_Exporter::build_debug_export()`.

| Key               | Type   | Description |
|-------------------|--------|-------------|
| section_key       | string | Section internal_key |
| group_key         | string | Deterministic group key |
| section_version   | string | Blueprint version |
| source            | string | `"registry"` |
| field_count       | int    | Number of fields in blueprint |
| registry_reference| string | e.g. `section_key:st01_hero` |

### 2.4 acf_mirror_diff_summary

Generated by `ACF_Field_Group_Debug_Exporter::build_diff_summary(registry_manifest, mirror_manifest)`.

| Key                     | Type  | Description |
|-------------------------|-------|-------------|
| in_registry_not_mirror  | list  | Group keys in registry but not in mirror |
| in_mirror_not_registry  | list  | Group keys in mirror but not in registry |
| version_mismatch        | list  | Objects: `group_key`, `registry_version`, `mirror_version` |
| summary                 | string| Human-readable one-line summary |

---

## 3. Mirror generation

- **Service:** `ACF_Local_JSON_Mirror_Service`.
- **Input:** Target directory path (e.g. export staging `acf_field_groups_mirror/`).
- **Output:** One JSON file per plugin-owned group: `{group_key}.json`. Contents are ACF-compatible field group arrays (key, title, fields, location, _aio_section_key, _aio_section_version). No live content values or secrets. A `manifest.json` in the same directory holds the acf_local_json_manifest.
- **Determinism:** Keys and structure are deterministic from registry blueprints; same blueprint set produces same mirror.

---

## 4. Export bundle inclusion

- **Optional category:** `acf_field_groups_mirror` (Export_Bundle_Schema::OPTIONAL_CATEGORIES).
- **When included:** Export_Generator writes mirror into staging `acf_field_groups_mirror/` (group JSON files + manifest.json) and includes that directory in the ZIP. Allowed in full operational backup and support bundle when the category is requested.
- **Restore/import:** Restore pipeline and import logic may treat `acf_field_groups_mirror` as reference/debug only; field groups are restored from registry/blueprint authority, not by re-importing mirror JSON as the primary source. Mirror can be used for comparison or recovery guidance.

---

## 5. Regeneration/repair integration

- After running ACF regeneration/repair (Prompt 222), operators may run mirror generation (e.g. via export with `acf_field_groups_mirror` or a dedicated capability-gated action) to refresh the mirror. Mirror generation is compatible with regeneration workflows; it does not replace them.
- Optional: regeneration service may accept a mirror service and a path to refresh the mirror after a successful repair (implementation-specific).

---

## 6. Security and permissions

- Only authorized technical/admin roles may generate or access debug exports. Mirror artifacts must not include secrets, live content values, or unsafe environment-specific data beyond approved field-definition metadata.
- Any export UI that includes the mirror category must be capability-gated and nonce-protected.

## 7. Example payloads

### 7.1 Example acf_local_json_manifest

```json
{
  "schema_version": "1",
  "mirror_version": "1",
  "generated_at": "2025-03-14T12:00:00Z",
  "plugin_version": "1.0.0",
  "source": "registry",
  "group_keys": ["group_aio_st01_hero", "group_aio_st05_faq"],
  "files": [
    { "group_key": "group_aio_st01_hero", "section_key": "st01_hero", "section_version": "1", "path_relative": "group_aio_st01_hero.json" },
    { "group_key": "group_aio_st05_faq", "section_key": "st05_faq", "section_version": "1", "path_relative": "group_aio_st05_faq.json" }
  ],
  "label": "ACF field groups mirror (internal/debug). Registry is source of truth."
}
```

### 7.2 Example field_group_debug_export_record

```json
{
  "section_key": "st01_hero",
  "group_key": "group_aio_st01_hero",
  "section_version": "1",
  "source": "registry",
  "field_count": 3,
  "registry_reference": "section_key:st01_hero"
}
```

### 7.3 Example acf_mirror_diff_summary

```json
{
  "in_registry_not_mirror": ["group_aio_st02_cta"],
  "in_mirror_not_registry": [],
  "version_mismatch": [
    { "group_key": "group_aio_st01_hero", "registry_version": "1", "mirror_version": "0" }
  ],
  "summary": "1 in sync, 1 only in registry, 0 only in mirror, 1 version mismatch."
}
```
