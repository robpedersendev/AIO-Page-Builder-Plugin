## 5. Recommended execution order

1. **SPR-001** — Bundle upload size (and optional MIME) limit.  
2. **SPR-002** — Verify Prompt_Experiments_Screen escaping (no code change if already done).  
3. **SPR-003** — Capability alignment (manage_options → plugin caps or documented exception).  
4. **SPR-004** — Privacy decision: register exporter/eraser or document “no personal data.”  
5. **SPR-011** — PrivatePluginBase legacy cleanup or isolation.  
6. **SPR-005, SPR-006** — Settings/Dashboard placeholder: implement or document deferred.  
7. **SPR-007** — Bundle confirm import: deferred; UI gated (preview only, link to Import/Export).  
8. **SPR-008, SPR-009, SPR-010** — Informational; docs only if needed.

## 6. Spec / decision-log notes

- **SPR-004 (privacy data audit):** Plugin stores personal data (WP privacy API sense). In scope: AI runs (actor), job queue (actor_ref), template compare user meta, bundle preview transient—exporter/eraser registered. Out of scope (not keyed by user): onboarding draft (site option), reporting log, industry profile/audit trail, diagnostics. Execution log table has actor_ref in schema but no writes in codebase; UI uses job queue. Decision: no code change. **Scope boundary (2025-03-18):** No expansion; boundary documented in [privacy-exporter-eraser-scope-boundary.md](privacy-exporter-eraser-scope-boundary.md) and decision in [privacy-exporter-eraser-scope-decision.md](privacy-exporter-eraser-scope-decision.md).  
- **SPR-005, SPR-006, SPR-007:** Remediation must not add features beyond the master spec; defer or document if spec does not define the behavior. **SPR-007:** As of 2025-03-18, bundle apply is in scope per [industry-bundle-apply-decision.md](industry-bundle-apply-decision.md); semantics defined by industry-pack-bundle-format-contract and industry-pack-import-conflict-contract. Until apply is implemented, UI remains preview-only with link to Import / Export. No confirm-import handler yet; Clear preview is nonce-protected.  
- **SPR-003:** No plugin-owned screen or widget retains `manage_options` by design. All in-scope screens use Capabilities constants: Crawler/Diagnostics → `VIEW_SENSITIVE_DIAGNOSTICS`; Onboarding → `RUN_ONBOARDING`; Settings → `MANAGE_SETTINGS`; Dashboard (root and provider) and Industry_Status_Summary_Widget → `VIEW_LOGS`. If any future screen must remain admin-only, add an explicit decision-log entry and in-code comment.
- **Import/Export ZIP size (optional hardening):** As of 2025-03-18, product decision: enforce explicit pre-move cap (50 MB) per [import-export-zip-size-limit-decision.md](import-export-zip-size-limit-decision.md). Implementation criteria in import-export-zip-size-limit-acceptance-criteria.md. Until implemented, current behavior (no pre-move cap) remains; closeout §3 listed this as optional.
- **SPR-008, SPR-009, SPR-010 (intentional placeholders):** These refer to intentional placeholders, shells, or schema-only artifacts per current spec. Stub_Execution_Handler (SPR-008), Tokens_Step_UI_Service (SPR-009), Profile_Snapshot_Data and provider `cost_placeholder` (SPR-010) are documented in code and ledger so future audits do not treat them as accidental implementation gaps. Do not treat as gaps unless the master spec later requires full implementation (token application, profile persistence, cost modeling, or new execution handlers).
